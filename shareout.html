ShareOut - UI Design - Native App

ShareOut is a charity giving app that allows users to donate regularly to the causes they care about. The native app takes the hassle out of manual donations and selects new charities for the user to support each month.

This project focussed on developing the concept, wire framing, understanding native design guidelines and prototyping.


Objective

Create a native charity giving app that is optimised for both iOS and Android.

Challenge

Make donating to various charities a simple, secure process.

Solution

Create an app that users can trust to donate to charities that serve their interests.

Approach

Define - Ideate - Design - Prototype

My Role

Complete ownership of UX and UI. Key deliverables for this project were proposal, user flows, competitive analysis, wireframes, prototypes and user testing

Define

User Research

I began by speaking with people about their experience of charitable giving. This allowed me to identify what users would like to be able to achieve through a charity app, and any frustrations they currently have.

Key Features

1


User will be able to set their monthly donation budget and interests/charity preferences

2


User will be able to easily view information about the charities they have donated to, both past and present

3


User will be notified of proposed charities each month and will be able to edit the selection if required

iOS Human Interface vs Material Guidelines

I wanted to make sure I created an app that provided optimum performance on each platform it was used. I therefore familiarised myself with both the iOS Human Interface Guidelines and the Material Guidelines so that features were designed specifically for each.

Competitive Analysis

UK Giving 2019 found that around 65% of people in the UK donate money to charity, however my research of the App Store and Google Play Store found very limited options for apps that help achieve this goal.

In order to research the features that my app would require, as well as what would and wouldnâ€™t work, I analysed Share the Meal and DonatePal as they offer similar concepts and functionalities.

I used this analysis to improve my low-fidelity wireframes as below:

1

I decided to add additional information to the landing page of the app so that the user feels more informed before signing up. This will also help with developing trust.

2

I also chose to amend the empty state home screen so that the decision to proceed is placed with the user instead of forced upon them.

3

For ease of use and interactivity it was decided to use a sliding scale for the user to decide how much they wish to donate.
